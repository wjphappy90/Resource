首先欢迎您们选择易宝支付提供的支付接入服务。此目录下面的例子是用ASP的写成，您可以直接把所有文件放在IIS应用的目录下，进行测试运行。
1）文件列表说明
	|-----------------------------------------------------
	|----index.html  							(提供给商户测试用的首页)
	|----onLine.html   						(提供给商户测试用的支付输入页面,提交后商户输入的参数会提交到易宝支付或相应银行网关.)
			|----req.asp					    (支付请求文件，通过此文件发起支付请求，商家可以在此文件中写入自己的订单信息等，然后把请求提交给易宝支付）
			|----callBack.asp				  (支付结果返回文件，通过此文件商家判断对应订单的支付成功状态，并且根据结果修改自己数据库中的订单状态）
	|----queryOrd.html   					(提供给商户测试用的订单查询输入页面)
			|----queryOrd.asp					(订单查询文件，通过此文件商家查询订单的支付状态）
	|----refundOrd.html   				(提供给商户测试用的订单退款请求输入页面)
			|----refundOrd.asp				(订单退款文件，通过此文件商家提交订单退款请求）
	|----yeepayCommon.asp   			(共通函数文件，不需要进行任何修改)
	|----merchantProperties.asp		(商家属性文件，商家可以在此文件中修改商户编号和密钥信息)

2）对于非虚拟主机用户我们提供了加密组件，只要注册后即可直接使用。
    需要把 lib 目录下面的文件COPY到WINDOWS系统目录\system32\ 目录下，然后直接注册或者运行regdll.bat问题即可。
	
3）商家测试可以先用易宝支付的测试商家测试成功再在merchantProperties.asp文件中修改成自己的商户编号和密钥信息。
  merchantId = "10000432521";
	keyValue = "8UPp0KE8sq73zVP370vko7C39403rtK1YwX40Td6irH216036H27Eb12792t";
	商户编号和密钥需要同时修改才有效！

4）请在req.asp文件中修改商户接收支付成功数据的地址。
	p8_Url = "http://localhost/HTMLcommon/callback.asp"
	商家正式运行时，必须把自己的服务器部署在公网上的服务器上，这样支付成功后易宝支付的服务器才能将支付结果及时返回给商家。
	为了商家技术工程师开发的方便，如果测试服务器在工程师自己的服务器上，可以把URL设定为localhost的方式，浏览器可以正式返回。
	
5）共通文件采用服务器包含的方式进行处理。
	如：
		<!-- #include file="merchantProperties.asp" -->
		<!-- #include file="yeepayCommon.asp" -->
	所以如果您修改共通文件请帮助每个文件能够编译通过。
	检查语法错误的最简单方式是用浏览器直接访问IIS中的此文件URL

6）易宝支付支持在商家选择银行和在易宝支付网关选择银行的两种方式。
	可以通过设定Req.asp中pd_FrpId的值来进行调整。
	如：pd_FrpId=""   					到易宝支付支付网关再选择银行
			pd_FrpId="1000000-NET"	直接进行易宝支付会员支付
			pd_FrpId="ICBC-NET"			直接进行中国工商银行在线支付
			pd_FrpId="SZX"					直接进行神州行卡在线支付
	其它银行编号请参考易宝支付的文档说明

7）对于网上下单的电话支付接口可以参考此文档进行开发。
